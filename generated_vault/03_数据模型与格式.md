# 数据模型与格式

本模块讨论矢量数据（点/线/面）、栅格数据、TIN 与点云的内部结构与存储格式，并引入现代云原生格式：GeoParquet、COG（Cloud Optimized GeoTIFF）、PMTiles、FlatGeobuf 等。

要点摘要：
- 矢量格式：Shapefile（已趋于弃用）、GeoPackage、FlatGeobuf、GeoJSON（适合小型数据或交互）
- 栅格格式：GeoTIFF / COG（用于云环境的优化栅格）
- 点云格式：LAS/LAZ、PDAL 处理流程
- 云原生格式优势：列式存储（GeoParquet）、切片/瓦片优化（PMTiles）、分块读取友好（COG）

## 格式选择决策（简要）
- 小文件/交互性：GeoJSON / FlatGeobuf
- 大规模矢量/分析：GeoParquet / PostGIS
- 大规模栅格/云发布：COG + STAC 描述
